# Worker на WSL2 - Інструкція

## ✅ Що вже працює:

1. Підключення до сервера ✅
2. Отримання завдань з черги ✅
3. Відправка результатів ✅

## ⚠️ Проблема з WSL2:

WSL2 не має нативного GUI, тому браузер не відкривається автоматично.

## 🔧 Рішення:

### Варіант 1: Використовувати Windows браузер (рекомендую)

**Ідея:** Worker працює в WSL2, але відкриває браузер у Windows

**Як:**
1. Worker отримує URL з черги
2. Відкриває браузер у Windows через `cmd.exe`
3. Ти тестуєш у Windows браузері
4. Натискаєш hotkey
5. Worker відправляє результат

**Створимо модифікований worker:**

```bash
cd ~/aiwork
nano worker_wsl.py
```

Або я можу створити його зараз.

---

### Варіант 2: Ручний режим (найпростіший для старту)

**Як працює:**
1. Worker показує URL в терміналі
2. Ти вручну відкриваєш браузер у Windows
3. Тестуєш
4. Натискаєш hotkey в терміналі WSL
5. Worker відправляє результат

**Переваги:**
- Не треба налаштовувати GUI
- Працює одразу
- Ти контролюєш процес

**Мінуси:**
- Треба вручну копіювати URL

---

### Варіант 3: WSLg (GUI в WSL2)

**Що це:** Офіційна підтримка GUI в WSL2 (Windows 11)

**Перевірка чи працює:**
```bash
echo $DISPLAY
# Має бути :0 або :1
```

У тебе є `:0` ✅

**Встановлення браузера:**
```bash
# Chromium в WSL
sudo apt install chromium-browser

# Або Firefox
sudo apt install firefox
```

**Проблема:** Може бути повільно, не всі скрінрідери працюють

---
