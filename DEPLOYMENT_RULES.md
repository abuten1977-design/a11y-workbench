# DEPLOYMENT CHECKLIST

## ⚠️ ПРАВИЛА ДЛЯ KIRO (AI Assistant)

### Перед любым деплоем ОБЯЗАТЕЛЬНО:

#### 1. ✅ Проверить какие файлы изменились
```bash
git status
git diff --name-only HEAD~1
```

#### 2. ✅ Проверить зависимости
Если менял:
- `api_server_v1.py` → проверь использует ли он `repository.py`, `database.py`, `exports.py`
- `repository.py` → деплой вместе с `api_server_v1.py`
- `database.py` → деплой вместе с `repository.py`
- `exports.py` → деплой вместе с `api_server_v1.py`

#### 3. ✅ Проверить схему базы данных
Если добавлял новые поля в таблицы:
- Проверить что на сервере есть эти колонки
- Создать миграцию если нужно

#### 4. ✅ Использовать deploy.sh
```bash
cd /home/butenhome/aiwork
./deploy.sh
```

НЕ деплоить файлы по одному вручную!

#### 5. ✅ После деплоя - запустить smoke test
```bash
TEST_URL=http://34.58.51.76:8000 node smoke_test.js
```

#### 6. ✅ Проверить логи после деплоя
```bash
gcloud compute ssh a11y-server --zone=us-central1-a --command="sudo journalctl -u a11y-api -n 20 --no-pager"
```

## ✅ ЧТО ДЕЛАТЬ ВСЕГДА:

1. **Коммит локально** → `git add` + `git commit`
2. **Проверить зависимости** → какие файлы связаны
3. **Запустить deploy.sh** → автоматический деплой всех файлов
4. **Проверить health** → curl /health
5. **Запустить smoke test** → node smoke_test.js
6. **Проверить логи** → journalctl
7. **Пуш в GitHub** → `git push`
