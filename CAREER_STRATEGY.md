## 💡 INTERVIEW PREPARATION

### Common Questions

**"Tell me about your accessibility testing experience"**
```
I'm a blind NVDA user who's turned my daily experience into professional expertise. I've tested 15+ digital products, identifying 200+ accessibility issues. I use Deque methodology and built my own testing system (A11y Workbench) to streamline workflow and generate professional reports. My unique value is combining real blind user perspective with technical skills and systematic approach.
```

**"How do you test accessibility?"**
```
I use manual testing with NVDA screen reader, following WCAG 2.2 guidelines. My process:
1. Understand user flows
2. Test with keyboard only
3. Navigate with NVDA
4. Document issues with evidence (screen reader output)
5. Map to WCAG criteria
6. Provide remediation recommendations
7. Retest after fixes

I also use structured checklists for consistency and A11y Workbench for documentation.
```

**"What's your biggest accessibility find?"**
```
[Prepare 2-3 stories from sample projects]
Example: "Found a keyboard trap in a modal dialog that completely blocked blind users from completing checkout. The modal couldn't be closed with keyboard, forcing users to reload the page. This was a WCAG 2.1.2 violation. I documented it with screen reader output evidence and recommended adding Escape key handler and focus management. After fix, checkout completion rate for screen reader users improved significantly."
```

**"Why should we hire you?"**
```
Three reasons:
1. Authentic perspective - I'm not simulating disability, I live it daily
2. Technical skills - I can communicate with developers, understand code, built my own testing system
3. Systematic approach - Deque-trained methodology, professional documentation, proven workflow

Plus, I'm genuinely passionate about making the web accessible. It's not just a job, it's personal.
```
