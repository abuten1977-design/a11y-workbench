## Що робить сервер (Google Cloud)?

### 1. Приймає запити 24/7

```python
# AI-агент відправляє:
POST http://твій_сервер_IP:8000/api/v1/check
{
  "url": "https://example.com",
  "priority": 1
}

# Сервер відповідає:
{
  "job_id": "abc123",
  "status": "pending",
  "queue_position": 3
}
```

### 2. Зберігає чергу

```
Черга завдань:
1. google.com - pending
2. github.com - pending  
3. wikipedia.org - processing (ти зараз тестуєш)
4. facebook.com - completed
```

### 3. Віддає результати

```python
# AI-агент перевіряє:
GET http://твій_сервер_IP:8000/api/v1/check/abc123

# Сервер відповідає:
{
  "status": "completed",
  "result": {
    "score": 85,
    "issues": ["keyboard navigation", "color contrast"]
  }
}
```

## Чому не можна без сервера?

### ❌ Варіант 1: Все на ноутбуці

**Проблема:**
- Твій ноутбук не працює 24/7
- Немає постійного IP
- AI-агент не може до тебе достукатись
- Якщо ноутбук вимкнений - втрачаєш клієнтів

### ❌ Варіант 2: Тільки платформи (Fable/UserTesting)

**Проблема:**
- Ти залежиш від них
- Вони беруть комісію 30-50%
- Не можеш масштабуватись
- Не можеш встановлювати свої ціни

### ✅ Варіант 3: Сервер + ноутбук (наш варіант)

**Переваги:**
- Сервер приймає запити 24/7
- Ти працюєш коли зручно
- Запити накопичуються в черзі
- Ти сам собі бос
- Можеш масштабувати (найняти інших)

## Що робить сервер (технічно)?

### Завдання сервера:

1. **HTTP API (FastAPI)**
   - Приймає POST запити
   - Віддає GET відповіді
   - Обробляє JSON

2. **Черга завдань (Redis)**
   - Зберігає список завдань
   - Відстежує статуси
   - Видаляє виконані

3. **Логування**
   - Хто відправив запит
   - Коли виконано
   - Скільки часу зайняло

4. **Webhook для платежів**
   - Приймає підтвердження оплати
   - Розблоковує завдання

### Що НЕ робить сервер:

- ❌ Не відкриває браузер
- ❌ Не запускає Playwright
- ❌ Не обробляє відео/зображення
- ❌ Не тренує AI моделі
- ❌ Не зберігає великі файли

**Аналогія:**
- Сервер = секретар в офісі (приймає дзвінки, записує в блокнот)
- Твій ноутбук = ти (робиш реальну роботу)

## Чому f1-micro достатньо?

### Параметри f1-micro:
- **CPU:** 1 vCPU (shared, burst до 100%)
- **RAM:** 0.6 GB (600 MB)
- **Disk:** 30 GB SSD
- **Network:** 1 Gbps
- **Ціна:** $0/міс (750 годин безкоштовно)

### Споживання ресурсів:

```
FastAPI + Uvicorn:  ~100 MB RAM
Redis:              ~50 MB RAM
Python:             ~100 MB RAM
OS (Ubuntu):        ~200 MB RAM
Залишок:            ~150 MB RAM (запас)
```

### Навантаження:

**10 запитів/день:**
- CPU: <5%
- RAM: ~400 MB
- Disk: <1 GB
- ✅ Відмінно працює

**50 запитів/день:**
- CPU: ~20%
- RAM: ~500 MB
- Disk: ~2 GB
- ✅ Нормально працює

**100+ запитів/день:**
- CPU: ~50-70%
- RAM: ~550 MB
- Disk: ~5 GB
- ⚠️ Може гальмувати, час апгрейдити

## Коли апгрейдити сервер?

### Сигнали для апгрейду:

1. **Response time >2 секунди**
   - Клієнти чекають довго
   - Втрачаєш запити

2. **CPU постійно >80%**
   - Сервер не встигає
   - Можуть бути помилки

3. **RAM постійно >90%**
   - Ризик crash
   - Повільна робота

4. **>100 запитів/день**
   - f1-micro не тягне
   - Час масштабувати

### Апгрейд до e2-micro:

```bash
# Зупинити
gcloud compute instances stop a11y-server --zone=us-central1-a

# Змінити тип
gcloud compute instances set-machine-type a11y-server \
  --machine-type=e2-micro \
  --zone=us-central1-a

# Запустити
gcloud compute instances start a11y-server --zone=us-central1-a
```

**Параметри e2-micro:**
- CPU: 2 vCPU (shared)
- RAM: 1 GB
- Ціна: ~$7/міс

**Коли варто:** Якщо заробляєш >$200/міс

## Альтернативи (чому вони гірші)

### Альтернатива 1: Ngrok (тунель до ноутбука)

```bash
# На ноутбуці
ngrok http 8000

# Отримаєш URL: https://abc123.ngrok.io
```

**Проблеми:**
- ❌ Ноутбук має бути онлайн 24/7
- ❌ URL змінюється при перезапуску
- ❌ Безкоштовний ngrok має ліміти (40 запитів/хв)
- ❌ Платний ngrok = $8/міс (дорожче за e2-micro)

**Висновок:** Google Cloud f1-micro краще

---

### Альтернатива 2: Потужний сервер (все на сервері)

**Параметри:**
- CPU: 4 vCPU
- RAM: 8 GB
- Disk: 50 GB SSD
- Ціна: ~$50-80/міс

**Що зможе:**
- Запускати браузер на сервері
- Автоматичні тести (без тебе)
- Обробляти 1000+ запитів/день

**Проблеми:**
- ❌ Дорого ($50-80/міс vs $0)
- ❌ Втрачаєш головну перевагу (реальний human validator)
- ❌ Конкуруєш з автоматичними інструментами
- ❌ Складніше налаштувати

**Коли має сенс:** Якщо заробляєш >$2000/міс і хочеш повну автоматизацію

---

### Альтернатива 3: Serverless (AWS Lambda, Google Cloud Functions)

**Ідея:** Платиш тільки за виконання функцій

**Проблеми:**
- ❌ Складно налаштувати чергу
- ❌ Cold start (перший запит повільний)
- ❌ Ліміти на час виконання (15 хв)
- ❌ Дорожче для постійного навантаження

**Коли має сенс:** Якщо <10 запитів/день (але тоді навіщо взагалі?)

## Фінальний вердикт

### f1-micro ($0/міс) + твій ноутбук = ідеально для старту

**Чому:**
1. ✅ Безкоштовно (Free Tier)
2. ✅ Сервер робить тільки те, що треба
3. ✅ Вся "важка" робота на твоєму ноутбуці
4. ✅ Легко масштабувати пізніше
5. ✅ Постійний IP і 24/7 доступність
6. ✅ Не залежиш від платформ

**Коли апгрейдити:**
- Якщо >100 запитів/день → e2-micro ($7/міс)
- Якщо хочеш найняти інших → e2-small ($14/міс)
- Якщо >500 запитів/день → e2-medium ($28/міс)

**Правило:** Апгрейдити тільки коли заробляєш >10× вартості сервера
