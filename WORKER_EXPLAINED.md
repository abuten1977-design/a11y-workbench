# Як працює Worker - Пояснення простими словами

## 🎯 Що таке Worker?

**Worker** = твій особистий помічник-робот на ноутбуці

**Що він робить:**
1. Підключається до сервера
2. Дивиться чи є завдання в черзі
3. Якщо є - бере перше
4. Відкриває браузер з сайтом
5. Чекає поки ти тестуєш
6. Коли ти натискаєш hotkey - відправляє результат
7. Повторює з наступним завданням

---

## 🔧 Технології і інструменти

### 1. Python 3 (мова програмування)
**Навіщо:** Проста мова для автоматизації

### 2. asyncio (асинхронність)
**Що це:** Можливість робити кілька речей одночасно
**Приклад:** 
- Чекати на твій hotkey
- Одночасно тримати браузер відкритим
- Одночасно слухати клавіатуру

**Аналогія:** Як ти можеш слухати музику і читати книгу одночасно

### 3. httpx (HTTP клієнт)
**Що це:** Бібліотека для спілкування з сервером
**Що робить:**
- Відправляє запити на сервер: "Дай мені завдання"
- Отримує відповіді: "Ось завдання: перевір google.com"
- Відправляє результати: "Перевірив, є проблеми"

**Аналогія:** Як поштар - носить листи туди-сюди

### 4. Playwright (автоматизація браузера)
**Що це:** Бібліотека для керування браузером
**Що робить:**
- Відкриває браузер (Chromium)
- Завантажує сайт
- Чекає поки завантажиться
- Закриває браузер після тестування

**Аналогія:** Як пульт від телевізора - вмикає, перемикає канали, вимикає

### 5. pynput (клавіатура)
**Що це:** Бібліотека для відстеження натискань клавіш
**Що робить:**
- Слухає коли ти натискаєш Ctrl+Alt+1/2/3
- Викликає відповідну функцію
- Працює навіть коли вікно не в фокусі

**Аналогія:** Як охоронець - стоїть і чекає сигналу

## 🔄 Як працює Worker (покроково)

### Крок 1: Запуск
```bash
python3 worker.py
```

**Що відбувається:**
1. Worker запускається
2. Перевіряє чи сервер доступний (http://34.58.51.76:8000)
3. Реєструє hotkeys (Ctrl+Alt+1/2/3/0)
4. Виводить: "Worker запущено, чекаю завдань..."

---

### Крок 2: Пошук завдань (кожні 10 секунд)

**Що робить:**
```python
# Worker відправляє запит на сервер
GET http://34.58.51.76:8000/api/v1/queue

# Сервер відповідає:
{
  "total": 2,
  "jobs": [
    {"job_id": "abc-123", "url": "https://google.com"},
    {"job_id": "def-456", "url": "https://github.com"}
  ]
}
```

**Якщо черга порожня:**
- Worker чекає 10 секунд
- Перевіряє знову
- Повторює поки не з'явиться завдання

**Якщо є завдання:**
- Бере перше з черги
- Переходить до Кроку 3

---

### Крок 3: Відкриття браузера

**Що робить:**
```python
# 1. Запускає Chromium
browser = await p.chromium.launch(headless=False)

# 2. Відкриває нову вкладку
page = await browser.new_page()

# 3. Завантажує сайт
await page.goto("https://google.com")
```

**Що ти бачиш:**
- Відкривається вікно браузера
- Завантажується сайт
- В консолі: "Сайт відкрито. Тестуй зі скрінрідером..."

---

### Крок 4: Очікування твоєї оцінки

**Що робить Worker:**
```python
# Входить в цикл очікування
while self.waiting:
    await asyncio.sleep(0.5)  # Чекає 0.5 сек
    # Перевіряє чи ти натиснув hotkey
    # Якщо так - виходить з циклу
```

**Що робиш ти:**
1. Запускаєш NVDA/JAWS
2. Тестуєш сайт:
   - Навігація клавіатурою (Tab, Enter, стрілки)
   - Чи читає скрінрідер елементи?
   - Чи логічний порядок?
   - Чи є alt-тексти на зображеннях?
   - Чи видимий фокус?
3. Приймаєш рішення:
   - Все добре → Ctrl+Alt+1
   - Є проблеми → Ctrl+Alt+2
   - Критичні помилки → Ctrl+Alt+3
   - Не можу перевірити → Ctrl+Alt+0

---

### Крок 5: Обробка hotkey

**Що відбувається коли натискаєш Ctrl+Alt+2:**

```python
def on_hotkey_issues(self):
    # 1. Створює результат
    self.result = {
        "status": "issues",
        "score": 60,
        "issues": ["keyboard_navigation", "focus_visible"],
        "message": "Виявлено проблеми з доступністю"
    }
    
    # 2. Зупиняє очікування
    self.waiting = False
    
    # 3. Виводить підтвердження
    print("⚠️ Відмічено: Є ПРОБЛЕМИ")
```

**Що далі:**
- Worker виходить з циклу очікування (Крок 4)
- Закриває браузер
- Переходить до Кроку 6

---

### Крок 6: Відправка результату

**Що робить:**
```python
# Відправляє POST запит на сервер
POST http://34.58.51.76:8000/api/v1/result/abc-123
{
  "status": "issues",
  "score": 60,
  "issues": ["keyboard_navigation", "focus_visible"],
  "message": "Виявлено проблеми з доступністю"
}

# Сервер відповідає:
{
  "message": "Result submitted",
  "job_id": "abc-123"
}
```

**Що ти бачиш:**
```
✅ Результат відправлено
💰 Заробив: $3.00
📊 Всього: 1 завдань, $3.00
⏱️ Час: 120 сек
```

---

### Крок 7: Повторення

Worker повертається до Кроку 2:
- Перевіряє чергу
- Якщо є ще завдання - бере наступне
- Якщо немає - чекає 10 секунд і перевіряє знову

## 🎮 Hotkeys (гарячі клавіші)

### Ctrl+Alt+1 - Добре ✅
**Коли використовувати:**
- Сайт повністю доступний
- Навігація клавіатурою працює
- Скрінрідер читає все правильно
- Немає критичних проблем

**Що відправляє:**
```json
{
  "status": "accessible",
  "score": 90,
  "issues": [],
  "message": "Сайт доступний, проблем не виявлено"
}
```

---

### Ctrl+Alt+2 - Є проблеми ⚠️
**Коли використовувати:**
- Є проблеми, але сайтом можна користуватись
- Наприклад: немає alt-текстів, погана структура заголовків
- Або: фокус не видимий, але навігація працює

**Що відправляє:**
```json
{
  "status": "issues",
  "score": 60,
  "issues": ["keyboard_navigation", "focus_visible", "aria_labels"],
  "message": "Виявлено проблеми з доступністю"
}
```

---

### Ctrl+Alt+3 - Критичні помилки ❌
**Коли використовувати:**
- Сайтом неможливо користуватись зі скрінрідером
- Наприклад: кнопки не натискаються клавіатурою
- Або: скрінрідер нічого не читає

**Що відправляє:**
```json
{
  "status": "critical",
  "score": 30,
  "issues": ["keyboard_navigation", "screen_reader", "color_contrast", "heading_structure"],
  "message": "Критичні проблеми з доступністю"
}
```

---

### Ctrl+Alt+0 - Пропустити 🚫
**Коли використовувати:**
- Сайт не завантажився
- Потрібна авторизація
- Технічна помилка

**Що відправляє:**
```json
{
  "status": "skipped",
  "score": 0,
  "issues": [],
  "message": "Не вдалося перевірити"
}
```

## 📊 Статистика

Worker веде статистику:
- **jobs_completed** - скільки завдань виконано
- **total_earned** - скільки заробив ($3 за завдання)
- **duration** - скільки часу зайняла перевірка

**Приклад:**
```
📊 Всього: 5 завдань, $15.00
⏱️ Середній час: 2.5 хв на завдання
💰 Погодинна ставка: ~$36/год
```

## 🛠️ Технічні деталі

### Асинхронність (async/await)

**Проблема без async:**
```python
# Відкрити браузер (блокує виконання)
browser.open()  # Чекаємо 2 сек

# Слухати клавіатуру (блокує виконання)
keyboard.listen()  # Чекаємо поки натиснуть

# НЕ МОЖНА робити одночасно!
```

**Рішення з async:**
```python
# Відкрити браузер (не блокує)
await browser.open()  # Запустили і пішли далі

# Слухати клавіатуру (не блокує)
keyboard.listen()  # Працює паралельно

# Чекати hotkey (не блокує)
while waiting:
    await asyncio.sleep(0.5)  # Перевіряємо кожні 0.5 сек
```

**Аналогія:**
- Без async: Ти можеш або варити каву, або читати книгу
- З async: Ти ставиш каву варитись і читаєш книгу поки вона готується

---

### HTTP запити (httpx)

**Що відбувається під капотом:**

```python
# 1. Worker створює HTTP запит
async with httpx.AsyncClient() as client:
    response = await client.get("http://34.58.51.76:8000/api/v1/queue")

# 2. Запит йде через інтернет до сервера
# 3. Сервер обробляє запит (читає файл черги)
# 4. Сервер відправляє відповідь назад
# 5. Worker отримує JSON

data = response.json()  # Перетворює JSON в Python dict
```

**Аналогія:**
- Worker = ти
- HTTP запит = телефонний дзвінок
- Сервер = друг
- JSON = інформація яку він тобі сказав

---

### Playwright (браузер)

**Що відбувається:**

```python
# 1. Запустити Chromium
browser = await p.chromium.launch(headless=False)
# headless=False означає "показати вікно"

# 2. Відкрити вкладку
page = await browser.new_page()

# 3. Завантажити сайт
await page.goto("https://google.com", timeout=30000)
# timeout=30000 = чекати максимум 30 секунд

# 4. Закрити
await browser.close()
```

**Чому Playwright, а не просто відкрити браузер?**
- Можна керувати програмно
- Можна чекати завантаження
- Можна обробляти помилки
- Можна автоматично закривати

---

### pynput (клавіатура)

**Як працює:**

```python
# Реєструємо hotkeys
with keyboard.GlobalHotKeys({
    '<ctrl>+<alt>+1': self.on_hotkey_good,
    '<ctrl>+<alt>+2': self.on_hotkey_issues,
}) as h:
    # Поки цей блок працює - hotkeys активні
    while True:
        # Робимо роботу
        pass
```

**Що відбувається:**
1. pynput слухає всі натискання клавіш в системі
2. Коли бачить Ctrl+Alt+1 - викликає функцію `on_hotkey_good()`
3. Функція змінює змінну `self.waiting = False`
4. Worker виходить з циклу очікування

**Аналогія:**
- pynput = охоронець біля дверей
- Hotkey = секретний стук
- Функція = що робити коли почув стук

## 💡 Чому саме така архітектура?

### Альтернатива 1: Все вручну
```
1. Відкрити браузер вручну
2. Скопіювати URL з сервера
3. Вставити в браузер
4. Тестувати
5. Відкрити термінал
6. Відправити curl запит з результатом
```
**Проблема:** Довго, нудно, багато кліків

---

### Альтернатива 2: Повна автоматизація
```
Worker сам тестує сайт без тебе
```
**Проблема:** AI не може замінити реального незрячого користувача

---

### Наше рішення: Напівавтоматизація ✅
```
Worker робить рутину (відкриває, відправляє)
Ти робиш експертну роботу (тестуєш, оцінюєш)
```
**Переваги:**
- Швидко (автоматизація рутини)
- Якісно (реальний експерт тестує)
- Зручно (один hotkey замість 10 кліків)
