### 🎯 Dashboard (http://34.58.51.76:8000/dashboard)

**Функції:**
- ✅ Перегляд черги завдань
- ✅ Кнопка "Взять задание" - відкриває сайт в новій вкладці
- ✅ Статистика (в черзі, виконано, заробіток)
- ✅ Автооновлення кожні 10 секунд
- ✅ Темна тема (доступна з NVDA)

**Режими оцінки:**

1. **Простий режим** (швидкі тести):
   - Alt+1 → ✅ Хорошо (good)
   - Alt+0 → ⏭️ Пропустить (skipped)

2. **Професійний режим** (детальні звіти):
   - Alt+2 → ⚠️ Проблемы → **відкривається форма**
   - Alt+3 → ❌ Критично → **відкривається форма**

**Форма детального звіту:**
```
┌─────────────────────────────────────┐
│ Опис проблеми: *                    │
│ [текстове поле]                     │
│                                     │
│ Елемент (опціонально):              │
│ [текстове поле]                     │
│                                     │
│ WCAG критерій (опціонально):        │
│ [dropdown: 87 критеріїв]            │
│                                     │
│ [Скасувати] [Відправити]            │
└─────────────────────────────────────┘
```

### 📚 WCAG База Знань
- **Джерело:** tenon-io/wcag-as-json (MIT License)
- **Версія:** WCAG 2.2
- **Критеріїв:** 87
- **Формат:** JSON з повною інформацією
- **Локація:** 
  - Повна база: `/home/butenhome/aiwork/wcag-data/wcag.json`
  - Спрощений список: `/home/butenhome/aiwork/wcag_criteria_simple.json`

**Приклад критерію:**
```json
{
  "id": "1.1.1",
  "title": "Non-text Content",
  "level": "A",
  "label": "1.1.1 Non-text Content (A)"
}
```


## 🔧 КОМАНДИ УПРАВЛІННЯ

### Локально
```bash
cd ~/aiwork
./status_server.sh      # Статус сервера
./restart_server.sh     # Перезапуск
```

### На сервері
```bash
# SSH підключення
gcloud compute ssh a11y-server --zone=us-central1-a

# Статус сервісу
sudo systemctl status a11y-api

# Перезапуск
sudo systemctl restart a11y-api

# Логи
sudo journalctl -u a11y-api -f

# Результати
cat /tmp/a11y_results.json | python3 -m json.tool
```

### Деплой
```bash
cd ~/aiwork
gcloud compute scp api_server_v2.py a11y-server:/tmp/api_server.py --zone=us-central1-a
gcloud compute ssh a11y-server --zone=us-central1-a --command="
    sudo cp /tmp/api_server.py /home/butenhome/a11y-api/api_server.py &&
    sudo chown butenhome:butenhome /home/butenhome/a11y-api/api_server.py &&
    sudo systemctl restart a11y-api
"
```
