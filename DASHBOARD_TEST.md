# 🧪 ИНСТРУКЦИЯ: Как тестировать Dashboard

## 🔧 ОТКЛЮЧИТЬ ПОДТВЕРЖДЕНИЕ ИНСТРУМЕНТОВ

```bash
kiro-cli settings chat.autoApproveTools true
```

Или при запуске:
```bash
kiro-cli chat --auto-approve-tools
```

## 🎯 ЧТО ТЕСТИРУЕМ

**Dashboard на сервере:** http://34.58.51.76:8000/dashboard

---

## 📋 ПОШАГОВЫЙ ТЕСТ

### 1. Открой Dashboard
```
http://34.58.51.76:8000/dashboard
```

### 2. Что должен увидеть:
- 📊 **Статистика:**
  - В очереди: 5 заданий
  - Выполнено: 0
  - Заработано: $0

- 📋 **Список заданий:**
  - GitHub.com
  - Wikipedia.org
  - StackOverflow.com
  - Reddit.com
  - Twitter.com

### 3. Взять задание:
1. Клик на кнопку **"Взять задание"** возле GitHub
2. Откроется **новая вкладка** с GitHub.com
3. На dashboard появится панель **"Текущее задание"**

### 4. Тестировать с NVDA:
1. Переключись на вкладку с GitHub (Alt+Tab)
2. Тестируй навигацию с NVDA:
   - Заголовки (H)
   - Ссылки (K)
   - Формы (F)
   - Landmarks (D)

### 5. Оценить:
1. Вернись на вкладку dashboard (Alt+Tab)
2. Нажми **Alt+1** (хорошо) или:
   - **Alt+2** - есть проблемы
   - **Alt+3** - критично
   - **Alt+0** - пропустить

### 6. Результат:
- ✅ Появится alert "Результат отправлен!"
- Статистика обновится: Выполнено: 1, Заработано: $5
- Следующее задание автоматически доступно
